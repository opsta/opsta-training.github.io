Workshop การจัดการเซิร์ฟเวอร์พันเครื่องด้วย Ansible รุ่นที่ 1

ในโลกขององค์กรแบบ DevOps ที่ธุรกิจสมัยใหม่จำเป็นจะต้องพร้อมรับกับการเปลี่ยนแปลงอย่างรวดเร็ว Ansible คือหนึ่งในเครื่องมือที่จะช่วยเพิ่มประสิทธิภาพ ลดค่าใช้จ่าย และลดความผิดพลาดในการจัดการระบบ ไม่ว่าจะเป็นการสร้าง การขยาย การดูแลจัดการ infrastructure และ application ตั้งแต่หนึ่งเครื่องไปจนถึงนับพันเครื่อง ทั้งแบบ traditional และบน cloud ที่ทำให้ง่าย สะดวก และรวดเร็วขึ้น

ในคอร์สนี้ ผู้เข้าร่วมอบรมจะได้ทำการเรียนรู้วิธีการสร้าง และใช้งาน Ansible สำหรับจัดการระบบ infrastructure ในองค์กร ไม่ว่าจะเป็น dedicated server, virtualization หรือระบบ cloud โดยผู้เข้าอบรมจะได้ลองเขียน Ansible ในการจัดการระบบจริงบน OpenStack Private Cloud ตั้งแต่พื้นฐาน ไปจนถึงการต่อยอดการใช้งาน playbook จากชุมชนทำให้ไม่ต้องเขียนตั้งแต่เริ่มต้น รวมถึงโจทย์และ best practice ที่มาจากประสบการณ์จริงกว่า 3 ปีของวิทยากร "เดียร์" จิรายุส นิ่มแสง ที่เป็นผู้สร้าง ร่วมพัฒนาและแจกจ่าย Ansible playbooks เป็นจำนวนมากให้กับชุมชน open source ทำให้หลังจากจบคอร์ส สามารถนำความรู้กลับไปใช้งานในองค์กรของคุณได้ทันที

คอร์สนี้เหมาะสำหรับ:
- IT Managers หรือ System Engineers ที่กำลังมองหาการทำ cloud automation และ configuration management ในการทำ infrastructure as a code เพื่อจัดการระบบและเซิร์ฟเวอร์ในองค์กร
- Developer หรือ DevOps ที่ต้องการศึกษาการทำ application deployment หรือการทำ service orchestration เพื่อสามารถสร้างระบบในการให้บริการได้รวดเร็วและแม่นยำยิ่งขึ้น
- บุคคลทั่วไปที่สนใจและต้องการศึกษาการทำ automation ตั้งแต่ระดับเบื้องต้น

สิ่งที่ต้องเตรียมสำหรับผู้เข้าอบรม
- มีความรู้พื้นฐาน Linux command line
- คอมพิวเตอร์ส่วนตัวที่ติดตั้ง Browser และ SSH Client เช่น Terminal หรือ Putty

สิ่งที่ผู้เข้าอบรมจะได้จากคอร์สนี้:
- รู้จักพื้นฐานและส่วนประกอบต่างๆ ของ Ansible
- รู้จักกับการวางโครงสร้าง แนวคิด และการเขียน roles อย่างมีประสิทธิภาพ
- รู้จักกับ Ansible Vault ในการปกป้องข้อมูลที่สำคัญ เช่น รหัสผ่าน
- ลองใช้งาน Ansible Galaxy role จากชุมชน และลองแก้ไขเพื่อต่อยอดการนำไปใช้งานต่อได้ โดยไม่ต้องเริ่มจากศูนย์
- ได้ลองใช้ Ansible playbooks ในการเชื่อมต่อกับระบบ OpenStack private cloud ในการสร้างเครื่องแบบ dynamic ที่หลายเซิร์ฟเวอร์ทำงานร่วมกัน
- รู้จักกับการเขียน Ansible playbooks แบบ best practice จากประสบการณ์จริงของวิทยากร
- รู้จักกับการใช้ Git version control ที่จะทำให้การทำ Infrastructure as a code ร่วมกันหลายๆ คนง่ายขึ้น
- มาตรฐาน Ansible Style Guide ที่ทำให้อ่านและดูแลโค้ดง่าย
- การวางโครงสร้างของ playbooks, variables, inventories และ roles เพื่อใช้ในการทำ infrastructure as a code แบบซับซ้อนในองค์กร
- Certificate รับรองว่าผ่านการอบรม OpenStack Administration จากบริษัท Opsta
- เสื้อที่ระลึกจากการอบรม

วิทยากร

จิรายุส นิ่มแสง (เดียร์)

  • จบการศึกษาระดับปริญญาตรี จากคณะเทคโนโลยีสารสนเทศ สถาบันเทคโนโลยีพระจอมเกล้าเจ้าคุณทหารลาดกระบัง
  • เป็นผู้พัฒนา Ansible playbooks ให้กับ Kaidee ที่ใช้สำหรับควบคุมทั้ง OpenStack และ Docker ในการสร้างระบบและ deploy แบบ zero-down-time ที่มีผู้ใช้เป็นจำนวนถึง 33 ล้านเพจวิวต่อวัน
  • มีประสบการณ์การเขียน Ansible มามากกว่า 3 ปี และเป็นผู้พัฒนาและแจกจ่าย Ansible playbooks และ roles ให้กับชุมชนเป็นจำนวนมาก
  • ได้รับชื่อว่าเป็นหนึ่งในผู้ร่วมพัฒนา OpenStack Ansible ซึ่งถือเป็น playbooks ที่ใหญ่และซับซ้อนมากที่สุดในโลก
  • เป็น speaker คอยเผยแพร่ความรู้ให้ชุมชนอยู่เสมอ
  • ปัจจุบันดำรงตำแหน่งเป็นผู้ก่อตั้งและ CEO บริษัท ออพซ์ตา (ประเทศไทย) จำกัด ซึ่งเป็นบริษัทที่มีทีมทำทางด้าน Cloud Automation ให้กับบริษัทต่างๆ มากมาย

- Ansible tips and techniques
- ลองใช้ Ansible สร้างเครื่องแบบ dynamic บน OpenStack
- ทำโจทย์ทดสอบความรู้และการรีวิวโค้ดจากวิทยากร
